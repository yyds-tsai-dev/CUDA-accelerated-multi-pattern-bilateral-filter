===== PAGE 11 =====
RISC-V Vector Extension (RVV)
’üĄVector ISA extension for the RISC-V architecture.
’üĄIntroduces vector registers to accelerate data-parallel workload
’üĄv0 ~ v31
’üĄEach vector register can contain multiple elements
’üĄv1 = [a1, a2, a3, a4, ...], v2 = [b1, b2, b3, b4, ...], ŌĆ”
’üĄA vector instruction can perform an arithmetic operation to all 
elements in parallel
’üĄE.g., vfmul.vv ’āĀ[a1*b1, a2*b2, a3*b3, a4*b4, ...]
11


===== PAGE 12 =====
RVV Instruction Naming Convention
’üĄRVV instructions usually follow <operation>.<format>
’üĄFor example, vfmul.vv,  vfadd.vf,  vadd.vx,  vredsum.vs
’üĄThe suffix describes the operand types
’üĄ.vv = Vector op Vector
’üĄ.vf = Vector op Scalar FP
’üĄ.vx = Vector op Scalar Integer
’üĄ.vi = Vector op Immediate
’üĄ.v = Vector load/store
’üĄ.vs = Vector to Scalar Reduction (e.g., summation)

===== PAGE 18 =====
Part 3 - SIMD-like RVV Parallelization
’üĄApply k-way loop unrolling.
’üĄFor every group of k iterations, we compute 
f (a1[1], b1[1]) + f (a1[2], b1[2]) + ŌĆ” + f (a1[n], b1[n])
f (a2[1], b2[1]) + f (a2[2], b2[2]) + ŌĆ” + f (a2[n], b2[n])
...
f (ak[1], bk[1]) + f (ak[2], bk[2]) + ŌĆ” + f (ak[n], bk[n])
’üĄChoose the block size k by yourself
’üĄIf your n is too large, you may partition the summation into blocks 
’üĄPerform across-k SIMD parallelization. Each vector lane corresponds to one 
iteration within a group.

===== PAGE 27 =====
CUDA programming
’üĄCUDA programs consist of
’üĄHost code: runs on CPU
’üĄDevice code (kernel): runs on GPU
’üĄCUDA Execution Hierarchy
’üĄThread: The smallest execution unit in CUDA.
’üĄWarp:
A group of 32 threads executed together in SIMT fashion.
’üĄBlock:
A group of threads that can synchronize and share shared memory.
’üĄGrid:
A collection of blocks launched by a kernel.
